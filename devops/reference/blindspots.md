# 盲区清单与询问协议

> AI无法通过SSH获取一切信息。这份文档定义了所有"不可自动获取"的关键信息领域，以及如何高效地向用户收集这些信息。

## 核心原则

```
能查的 → 自己查，不浪费用户时间
不能查的 → 立即问，不瞎猜
问的时候 → 告诉用户去哪看、看什么、截图什么
```

**反模式（绝对禁止）：**
- ❌ 服务器上查不到原因，就猜"可能是XXX"然后让用户自己试
- ❌ 反复在服务器上折腾，忽略云平台层面的可能性
- ❌ 问用户"能不能检查一下"但不说去哪里检查
- ❌ 把所有可能的盲区一次性甩给用户（应该按排查逻辑逐步询问）

## 一、云平台安全组 / 防火墙

### 什么时候问
- 端口在服务器上已监听（`ss -tlnp` 确认），但外部无法访问
- `curl localhost:PORT` 正常，但 `curl EXTERNAL_IP:PORT` 超时
- 新部署的服务外部无法访问
- 切换了服务器端口后外部访问失败

### 怎么问
```
我在服务器上确认了端口 [PORT] 已经正常监听，服务在本地可以访问。
但从外部无法连接，这通常是**云平台安全组**没有放行该端口。

请帮我检查：
→ 登录 [阿里云/腾讯云/华为云] 控制台
→ 找到这台服务器的实例
→ 进入「安全组」配置
→ 确认是否有放行 [PORT] 端口（TCP 入方向）的规则

如果没有，需要添加一条规则：
  协议: TCP
  端口: [PORT]
  来源: 0.0.0.0/0（或你需要的IP范围）
  策略: 允许
```

### 各云平台路径速查
| 云平台 | 安全组入口 |
|--------|-----------|
| 阿里云 | 控制台 → ECS → 实例详情 → 安全组 → 配置规则 |
| 腾讯云 | 控制台 → CVM → 实例详情 → 安全组 → 入站规则 |
| 华为云 | 控制台 → ECS → 实例详情 → 安全组 → 入方向规则 |

## 二、ICP 备案（中国大陆服务器特有）

### 什么时候问
- 域名解析到中国大陆服务器，但访问返回空白页/拦截页/备案提示页
- 新域名绑定到大陆服务器后无法访问
- 域名在A云厂商备案，但解析到B云厂商的服务器
- 网站突然被阻断，之前一直正常

### 备案问题分类

#### 2.1 未备案
```
域名 [domain] 解析到了大陆服务器 [IP]，但可能尚未完成ICP备案。
中国大陆的服务器要求域名必须完成工信部ICP备案才能正常访问。

请确认：
→ 该域名是否已在工信部完成ICP备案？
→ 可以在 https://beian.miit.gov.cn/ 查询

如果未备案，有两个选项：
1. 办理ICP备案（需要 1-20 个工作日）
2. 将服务迁移到境外服务器（如香港、新加坡），不需要备案
```

#### 2.2 已备案但未接入当前服务商
```
域名 [domain] 可能已在工信部备案，但需要在当前服务器提供商处完成「备案接入」。

举例：域名在阿里云备案，但现在解析到了腾讯云的服务器 → 需要在腾讯云做备案接入。

请确认：
→ 域名的ICP备案是在哪个服务商办理的？
→ 当前服务器是哪个云平台的？
→ 如果不一致，需要在当前云平台做「接入备案」

各平台备案接入入口：
  阿里云：https://beian.aliyun.com/
  腾讯云：https://cloud.tencent.com/product/ba
  华为云：https://beian.huaweicloud.com/
```

#### 2.3 备案被取消/注销
```
之前正常的网站突然被阻断，可能是备案被取消了。常见原因：
- 备案信息不准确（主体信息变更未更新）
- 网站内容与备案信息不符
- 网站存在违规内容
- 备案年审/核验未通过
- 主体（公司/个人）信息变更未同步

请检查：
→ 登录 https://beian.miit.gov.cn/ 查询域名备案状态
→ 登录云平台备案管理页面查看是否有异常通知
→ 检查注册邮箱/手机是否收到工信部或云平台的备案相关通知
```

#### 2.4 经营性备案 / 特殊行业
```
如果网站涉及以下业务，可能需要额外资质：
- 在线交易/电商 → 需要 ICP 经营许可证（不只是备案）
- 新闻/出版 → 需要前置审批
- 医疗/药品 → 需要相关行业许可
- 教育/培训 → 部分地区需要教育部门审批
- 视频/直播 → 需要网络视听许可证

请确认网站业务类型，我判断是否需要额外资质。
```

## 三、域名与 DNS

### 什么时候问
- `dig` / `nslookup` 解析结果不符合预期
- 域名解析到了错误的IP
- DNS变更后长时间未生效
- 域名无法解析（NXDOMAIN / SERVFAIL）

### 怎么问
```
DNS解析显示 [domain] 指向 [IP]，但预期应该指向 [expected_IP]。

请确认：
→ 登录域名注册商（阿里云/腾讯云/Cloudflare/GoDaddy等）的DNS管理页面
→ 检查 [domain] 的 A 记录（或 CNAME 记录）指向是否正确
→ 确认 DNS 是否在其他地方被覆盖（比如 CDN、Cloudflare 代理等）

需要我知道：
1. 域名在哪个平台注册/管理的？
2. DNS 解析在哪里配置的？（可能和注册商不是同一个）
3. 是否使用了 CDN 或 Cloudflare？
```

### DNS 变更未生效
```
DNS 变更可能需要时间传播。

当前 TTL 值是 [X] 秒，理论上最长需要 [X/60] 分钟。
但实际上某些ISP的DNS缓存可能更久。

请确认：
→ DNS 变更是什么时候做的？
→ 可以尝试用不同DNS检查：
  - https://dnschecker.org/ 查看全球DNS传播状态
  - 或本地用 nslookup [domain] 8.8.8.8 测试Google DNS
```

## 四、SSL/TLS 证书

### 什么时候问
- 证书错误但服务器上找不到证书文件
- 使用了CDN或云平台托管的证书
- 需要确认证书是在哪里申请/管理的
- 证书续期失败

### 怎么问
```
SSL 证书出现问题：[具体错误]

请确认：
→ 证书是在哪里申请的？（Let's Encrypt / 阿里云 / 腾讯云 / 其他CA）
→ 证书是在服务器上管理，还是通过云平台/CDN托管？
→ 如果通过云平台：登录控制台 → SSL证书管理 → 查看证书状态和过期时间
→ 如果使用CDN：CDN控制台 → HTTPS配置 → 查看证书绑定状态
```

## 五、CDN 配置

### 什么时候问
- 源站正常但用户访问异常（缓存问题）
- 域名 CNAME 指向了 CDN 地址
- HTTPS 在源站正常但通过CDN访问报错
- 访问返回的内容是旧版本（CDN缓存）

### 怎么问
```
从 DNS 记录来看，[domain] 通过 CDN 加速。CDN 可能有自己的缓存和配置。

请帮我确认：
→ CDN 是哪家的？（阿里云CDN / 腾讯云CDN / Cloudflare / 其他）
→ 登录 CDN 控制台检查：
  1. 回源地址是否正确（应该指向 [expected_origin]）
  2. 缓存规则是否合理
  3. HTTPS 配置是否正确（证书是否有效）
  4. 是否有访问控制规则（IP黑白名单、Referer限制等）

如果是缓存问题，可以尝试：
→ CDN 控制台 → 刷新缓存/预热 → 输入需要刷新的 URL
```

## 六、云平台特殊限制

### 6.1 带宽/流量限制
```
服务器访问很慢，但服务器本身CPU/内存/磁盘都正常。

可能是云平台的**带宽限制**到了上限。

请检查：
→ 控制台 → 实例详情 → 查看带宽配置（公网带宽是多少Mbps？）
→ 控制台 → 监控 → 查看网络流量图表（是否达到带宽上限？）
→ 是否是按量付费带宽？余额是否充足？
```

### 6.2 实例到期/欠费
```
服务器突然无法访问，所有服务都不通。

可能是云平台实例**到期或欠费**被停机。

请检查：
→ 登录云平台控制台
→ 查看实例状态（运行中 / 已停止 / 已过期）
→ 查看账户余额和欠费通知
→ 检查注册邮箱/手机是否有续费提醒
```

### 6.3 内容合规拦截
```
网站之前正常，突然被阻断或显示违规提示页面。

可能是**内容审查**触发了阻断。常见情形：
- 网站内容与备案信息不符（备案为个人博客，实际是商城）
- 网站存在未经授权的内容
- 被举报后触发审查
- 云平台自动扫描发现违规内容

请检查：
→ 云平台控制台是否有「安全中心」或「内容审查」的告警通知
→ 注册邮箱/手机是否收到整改通知
→ 登录工信部网站 https://beian.miit.gov.cn/ 查看备案状态是否正常
→ 确认网站当前展示的内容是否与备案信息一致
```

### 6.4 DDoS 防护触发
```
服务器突然无法访问，但其他网络都正常。

可能触发了云平台的 **DDoS 防护/黑洞机制**。
当检测到对服务器的大量流量攻击时，云平台会暂时停止该IP的所有公网访问（进入"黑洞"）。

请检查：
→ 控制台 → 安全中心 / DDoS防护 → 查看是否有黑洞/清洗记录
→ 黑洞通常持续 30分钟 - 24小时（各平台不同）
→ 无法手动解除，只能等待自动恢复，或购买DDoS高防服务
```

## 七、第三方服务

### 7.1 邮件服务
```
邮件发送失败，但代码和SMTP配置看起来正确。

云服务器默认可能**封锁了25端口**（防垃圾邮件）。

请确认：
→ 是否在云平台申请过"25端口解封"？
→ 大多数云平台默认封锁25端口，需要提工单申请
→ 替代方案：使用 465(SSL) 或 587(STARTTLS) 端口，或使用邮件API服务
```

### 7.2 对象存储（OSS/COS/OBS）
```
上传/下载文件失败，程序报权限或跨域错误。

请确认：
→ 对象存储的 AccessKey/SecretKey 是否正确？
→ Bucket 的权限设置（公读/私有）
→ CORS 配置（如果是浏览器端直传）
→ Bucket 所在区域是否和服务器在同一区域（跨区域可能有延迟或限制）
```

### 7.3 API密钥/配额
```
第三方API调用失败，返回 401/403/429 错误。

请确认：
→ API Key 是否过期或被吊销？
→ 账户余额是否充足？（很多API按量计费）
→ 是否触发了速率限制（Rate Limit）？
→ API Key 的权限范围是否包含当前调用的接口？
```

## 八、询问协议

### 询问时机判断

```
技术排查 → 结果正常 → 但问题依然存在 → 进入盲区排查
                                           ↓
                                   识别最可能的盲区类型
                                           ↓
                                   向用户询问（具体、可操作）
```

### 询问格式规范

**标准格式：**
```
🔍 需要确认以下信息（我无法在服务器上直接获取）：

**背景**：我已经在服务器上排查了 [已查内容]，结果 [结果描述]，但问题仍然存在。

**需要你帮忙检查**：
→ 登录 [具体平台] 的 [具体页面]
→ 查看 [具体字段/配置]
→ 告诉我 [需要什么信息]

**为什么需要这个信息**：[解释这个信息如何帮助诊断]
```

### 分步询问原则

**不要一次问太多。** 按排查逻辑逐步询问：

1. 先问最可能的盲区（根据已有证据判断）
2. 得到回答后，如果排除了该可能性，再问下一个
3. 每次最多问 2-3 个相关联的问题

**反例（不要这样做）：**
```
请帮我检查：安全组、备案状态、DNS配置、CDN设置、SSL证书、带宽限制、账户余额...
```

**正例：**
```
从证据来看，端口在服务器上正常监听，但外部无法连接。最大可能是安全组配置。

请帮我检查阿里云控制台 → ECS实例 → 安全组，确认端口 443 是否放行了入方向规则。
```

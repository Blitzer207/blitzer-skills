GetStimulationPackage(*[returnParamSets](#ApiClient.ReportPackage.GetStimulationPackage.returnParamSets "ApiClient.ReportPackage.GetStimulationPackage.returnParamSets (Python parameter) — Specifies, how to deal with stimulation parameter sets.")=`True`*)[¶](#ApiClient.ReportPackage.GetStimulationPackage "Link to this definition")  
This call is only valid for analysis packages and returns the stimulation package.

Note:  
For older reports (before 2022.2) this method may return None if no trace analysis was executed.

Parameters:  returnParamSets : bool[¶](#ApiClient.ReportPackage.GetStimulationPackage.returnParamSets "Permalink to this definition")  
Specifies, how to deal with stimulation parameter sets. If True, the parameter set that has been used to stimulate the execution of the analysis package is returned.

If False, the stimulation package itself is returned, which is of type [`ReportParameterizedPackage`](ReportParameterizedPackage.md#ApiClient.ReportParameterizedPackage "ApiClient.ReportParameterizedPackage (Python class) — Call this method if you want to use API-objects in multiple contexts, e.g. if you want to insert a specific test step in multiple blocks.") for parameterized stimulations.

Returns:  The stimulation package. The default return value will be of type [`ReportPackage`](#ApiClient.ReportPackage "ApiClient.ReportPackage (Python class) — Call this method if you want to use API-objects in multiple contexts, e.g. if you want to insert a specific test step in multiple blocks."). If returnParamSets is False, it is possibly also of type [`ReportParameterizedPackage`](ReportParameterizedPackage.md#ApiClient.ReportParameterizedPackage "ApiClient.ReportParameterizedPackage (Python class) — Call this method if you want to use API-objects in multiple contexts, e.g. if you want to insert a specific test step in multiple blocks.").

Return type:  [`ReportProjectElement`](ReportProjectElement.md#ApiClient.ReportProjectElement "ApiClient.ReportProjectElement (Python class) — Call this method if you want to use API-objects in multiple contexts, e.g. if you want to insert a specific test step in multiple blocks.")
